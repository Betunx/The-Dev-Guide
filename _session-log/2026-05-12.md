**Estado mental:** explorando, primera vez con varias herramientas

## Tecnos nuevas
- **GitHub CLI (`gh`)** — alternativa a entrar a github.com para crear repos, autenticarte, etc.
- **Personal Access Token (PAT)** — la forma de autenticarte en GitHub vía HTTPS desde que removieron password auth en 2021.
- **Claude Code (VSCode)** — para asistirme con docs y organización del repo.

## Comandos / atajos que voy a querer volver a usar
- `git remote add origin <url>` — conectar repo local a un remoto.
- `git branch -M main` — renombrar la rama actual a `main`.
- `git push -u origin main` — push inicial y dejar tracking configurado.

## Errores que me costaron y cómo los resolví
- ❌ `error: src refspec main does not match any` → no había ningún commit aún. `git add .` solo no basta, falta `git commit`. **Aprendizaje:** el orden es siempre `add → commit → push`.
- ❌ `error: remote origin already exists` → ya había corrido `git remote add origin` antes. Para arreglar: `git remote set-url origin <url>` en lugar de `add`.
- ❌ Corrí `git commit` antes de `git add` → no commiteó nada porque no había nada staged.

## Notas sueltas
- El PAT solo se ve **una vez** al crearlo, hay que copiarlo de inmediato.
- Permisos mínimos en el PAT = solo `repo`.
- Windows guarda credenciales en el Administrador de credenciales; ahí se borran si hay problemas.
